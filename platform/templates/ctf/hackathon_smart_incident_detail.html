Scenario Nova Systems

Estimer les incidents reels à partir des activités suspectes

Nova Systems recoit chaque jour des alertes de sécurité. Certaines alertes correspondent a de vrais incidents, d'autres sont seulement des activités suspectes sans gravite. Le centre de supervision veut un petit service web intelligent capable d'estimer rapidement le nombre d'incidents confirmes à partir du nombre d'activités suspectes détectées.

Votre mission est de comprendre les données, construire un modèle de régression linéaire simple, expliquer son fonctionnement en pseudo-code, puis imaginer comment ce modèle serait utilisé dans une interface web.

Donnee d'entree x : suspicious_activities. Donnee de sortie y : confirmed_incidents. Type d'apprentissage : supervisé.

{% include "ctf/partials/leaderboard.html" %}

Objectif

Ce que votre solution doit faire

  • Lire un petit dataset de 15 exemples.
  • Identifier clairement l'entree x et la sortie y.
  • Observer si une relation existe entre les deux variables.
  • Construire une régression linéaire.
  • Utilisér l'équation du modèle pour faire des prédictions.
  • Imaginer une page web avec un champ de saisie et un resultat.

Équipes

Organisation conseillee

  • 4 a 5 participants par équipe.
  • Un coordinateur.
  • Un responsable données et calculs.
  • Un responsable pseudo-code.
  • Un responsable maquette web et réseau.
  • Un responsable presentation.

Dataset

Fichier fourni

Le fichier smart_incident_train.csv contient 15 anciennes observations de Nova Systems. Chaque ligne represente une periode d'activité surveillee par le SOC.

ColonneSignificationRole dans le modèle
sample_idIdentifiant de l'observationRepere, pas une variable de calcul
suspicious_activitiesNombre d'activités suspectes détectéesEntree x
confirmed_incidentsNombre d'incidents confirmes après analyseSortie y a prédire

Pendant l'epreuve, le jury donnera trois nouvelles valeurs de suspicious_activities. Vous devrez utilisér votre équation pour estimer les incidents confirmes correspondants.

Travail attendu

Les 6 étapes de votre équipe

1. Comprendre le problème

Reformulez le besoin avec vos mots : qui utilisé le service, quelle valeur est entree, quelle valeur est predite et pourquoi cette estimation aide Nova Systems.

2. Analyser le dataset

Identifiez x et y, observez l'evolution des valeurs, faites un petit graphique et dites si la relation semble presque lineaire.

3. Construire le modèle

Calculez les paramêtres de la droite de regression. Les formules utiles seront fournies par l'encadrement le jour de l'activité.

4. Ecrire le pseudo-code

Expliquez les étapes de calcul sans entrer dans un langage precis : lire x, appliquér l'équation, afficher la prediction.

5. Imaginer l'interface web

Dessinez une page simple : un titre, un champ pour saisir le nombre d'activités suspectes, un bouton Estimer et une zone de resultat.

6. Expliquer le réseau

Montrez le trajet de l'information : utilisateur, navigateur, requété vers le serveur, calcul du modèle, reponse affichee dans la page.

Livrable final

Archive ZIP attendue

Contenu attendu dans l'archive :

  • Un fichier de pseudo-code.
  • La maquette de l'interface web.
  • Le schéma client-serveur.
  • Un document texte, au format libre, qui contient les informations ci-dessous.

Le document texte doit preciser :

  • Nom du projet.
  • Definition du problème.
  • Identification de x et y.
  • Type d'apprentissage utilisé.
  • Équation finale du modèle.
  • Prédictions demandees par le jury.
  • Limites du modèle.
  • Noms et rôles des membres.
